# Опис проєкту

Коротко: це навчальний/демонстраційний проєкт для рендерингу простих 2D-об'єктів поверх OpenGL контексту, з підтримкою текстур, простих колайдерів і UI через ImGui.

## Структура
- `src/main.c` — точка входу, приклад роботи з API (створення вікна, об'єктів, головний цикл).
- `src/glad/` — реалізація GLAD (завантажувач функцій OpenGL).
- `src/lib/render/` — рендеринг, управління об'єктами, шейдери, колайдери.
- `src/lib/UI/` — оболонка над ImGui для простого виклику з C.
- `src/lib/win/` — обгортки для створення вікна на базі GLFW.

## Можливості
- Рендеринг текстур або суцільного кольору за допомогою простого шейдера.
- Автоматичне визначення кордонів колайдера по вершинах при створенні об'єкта.
- Перевірка зіткнень, тригери, рух об'єктів і прозорість.
- ImGui для відладкових вікон і оверлею.
- Локальні вихідні файли GLAD та ImGui включені до репозиторію.

## Як збирати

### Основна збірка

```bash
mkdir -p build && cd build
cmake .. -DBUILD_APP=ON -DAPP_ENTRY_FILE=src/main.c
cmake --build . --config Release
```

### Збірка конкретного файлу як entry point

```bash
cmake -S . -B build -DBUILD_APP=ON -DAPP_ENTRY_FILE=src/hsgdf.c
cmake --build build
```

### Збірка тільки бібліотеки

```bash
cmake -S . -B build -DBUILD_APP=OFF
cmake --build build
```

### Збірка GLFW із вихідників

Якщо на машині не встановлено GLFW, або ти хочеш використовувати локальну збірку бібліотеки, додай:

```bash
cmake -S . -B build -DGLFW_FROM_SOURCE=ON
cmake --build build
```

### Збірка на Windows

```powershell
mkdir build; cd build
cmake .. -A x64 -DBUILD_APP=ON -DAPP_ENTRY_FILE=src\main.c -DGLFW_FROM_SOURCE=ON
cmake --build . --config Release
```

## Опції CMake

- `BUILD_APP` — якщо `ON`, створюється виконуваний файл `app`.
- `APP_ENTRY_FILE` — C-файл, який компілюється як точка входу для програми.
- `GLFW_FROM_SOURCE` — завантаження та побудова GLFW із вихідників, якщо системного пакету немає або потрібен вбудований збірка.

## Використання як бібліотеки

Проєкт будує статичну бібліотеку `libapp` і створює CMake-аліас `game::libapp`.
Інші проєкти можуть лінкувати цю бібліотеку та використовувати API з репозиторію.
